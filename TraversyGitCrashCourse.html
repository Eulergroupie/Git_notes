Git Notes from Crash Course by Traversy

Basic Commands

  • git init: initializes repository and workspace
  • git add file: saves workspace files to staging
  • git commit: saves staged files to the local repository
  • git status: shows workspace versus staging info
  • git push: saves local repository to a remote repository
  • git pull: updates local repository from remote repository
  • git clone: copies a remote repository to the current folder

Getting Started

git --version Tells you if you have git and what version if you do have it.
download here if you need it
Git Bash is a highly recommended alternative to the command prompt for dealing with your repositories.
git config --global user.name "your name" to indicate who makes changes.
git config --global user.email "your email" so you can be contacted. git config --list will show git configuration environment variables.

An example of a normal workflow

Make a new folder for each project.
After creating some files, you find that you want to save them to your repository.
Type git init which will write a hidden file called .git
Type git add filename to add your file to staging.
Type git status to show what is staged and what is not.
If you decide to remove a file from staging, type git rm --cached filename
Wildcards can be used to describe filenames, and git add . is a fast way to add everything in the folder.
git commit will open up your selected editor (shown in environment variables) to make a comment before saving everything in the staging area to your local repository.
git commit -m "comment" is generally more convenient.

You can list files that you don't want included in staging commands in a file that you create called .gitignore. Files that are created during the compilation process are good candidates for the ignore file. git add . can then be used without pulling those ignored files into staging.

master is the default name of your local repository's top level. You can branch from the master in order to experiment or make changes that will require review/testing before they are included in master.
git branch branch-name will create a new branch from the current one (master at first).
git checkout branch-name changes your current branch to the new one (like cd)
Changes and new files committed will be a part of the branch that you are in. A file manager will show the new files. If you switch back to master with git checkout master, changes and new files disappear from view (shown in file manager and any open files in editors).
If you now type git merge branch-name from the master branch, the changes will be brought into the master branch. git branch -d branch-name then deletes the branch.

Going remote

Create an account on github if you haven't already. Use their UI to create a new repository and copy the url for it to your clipboard. There are good instructions in the UI for creating the remote repository on the command line.
The important part is git remote add origin https://github.com/account-name/repo-name.git
The name of your remote's master branch is usually origin. Typing git remote will show the current remote branch name.
Type git push -u origin master to save your local repo to the remote.
A README.md (md for markdown) is standard for describing your project to the public. Once created...
git add README.md
git commit -m "added README"
git push

Starting from someone's project

Change directories on your computer to the location that you would put a project directory for the desired project.
Copy the url for the project in GitHub to your clipboard. On the command line type git clone copied url
If you are working on it and believe that someone else may have made a change (or right before you try to upload a change) type git pull to update.
