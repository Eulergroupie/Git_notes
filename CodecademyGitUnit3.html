Codecademy Git Unit 3 Notes

Collaboration

Git offers a suite of collaboration tools to make working with others on a project easier. A remote repository is a shared Git repository that allows multiple collaborators to work on the same Git project from different locations. Collaborators work on the project independently, and merge changes together when they are ready to do so.

Features of a remote repository:
  • A complete replica of the project on your own computers
  • A way to keep track of and review each other’s work
  • Access to a definitive project version

Cloning

In order to get a replica of a repository type git clone remote_location clone_name.
remote_location tells Git where to go to find the remote repository. This could be a web address or a filepath.
clone_name is the name you give to the directory in which Git will clone the repository locally. Your local repository is not visible to other collaborators. The default name of the remote repository is origin.
Type git remote -v to see a list of a Git project's remotes.

Fetching and Merging Updates

An easy way to see if changes have been made to the remote and bring the changes down to your local copy is with git fetch. This command will not merge changes from the remote into your local repository. It brings those changes onto what’s called a remote branch. Even though the new commits have been fetched to your local copy of the Git project, those commits are on the origin/master branch. Your local master branch has not been updated yet, so you can’t view or make changes to any of the work added. Now we would use the git merge command to integrate origin/master into your local master branch. Type git merge origin/master to do this. Git will perform a “fast-forward” merge, bringing your local master branch up to speed with the most recent commit on the remote.

Git Workflow

The workflow for Git collaborations typically follows this order:

  1. Fetch and merge changes from the remote
  2. Create a branch to work on a new project feature
  3. Develop the feature on your branch and commit your work
  4. Fetch and merge from the remote again (in case new commits were made while you were working)
  5. Push your branch up to the remote for review

Step 5 is where you share your work with the remote repository by typing git push origin your_branch_name, where your_branch_name is the local branch that you worked on and wish to update the remote.

Git projects are usually managed on Github, a website that hosts Git projects for millions of users. With Github you can access your projects from anywhere in the world by using the basic workflow you learned here.
